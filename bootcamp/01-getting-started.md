# Module 1: Getting Started

## Just Start Asking

The most common reason faculty don't get value from AI is not lack of skill — it's waiting until they feel ready. There is no ready. Literacy develops through actual use, not through preparation for use.

The simplest framing: treat AI like a search engine that talks back. You don't need formal questions. You don't need technical language. You just need to start.

## Four Prompts to Start With

These cover the range of useful starting points for faculty.

**You have a technical question you'd normally Google:**
> "Why does video quality drop when I record in Teams? I want to send recorded lectures to students without the file size being huge."

**You have a task in your LMS you can't figure out:**
> "How do I export the event log from D2L so I can see when students accessed a specific page?"

**You have a document and want to do something with it:**
> "Here's an assignment prompt I've been using for three years. [paste it] What's unclear about the instructions? What would a first-generation student find confusing?"

**You don't know exactly what you want yet:**
> "I'm rethinking a research methods assignment. I'm not sure what I want to change. Ask me questions until you understand what I'm trying to assess."

That last one is often the most productive. AI is genuinely useful for helping you figure out what you actually want before you've articulated it.

## When the Response Isn't What You Wanted

If a response feels generic, off-target, or too long — don't start over. Just redirect in the same conversation:

- *"That's not what I meant. What I actually need is..."*
- *"Too long. Give me the same answer in two sentences."*
- *"You're assuming I have background in this. I don't. Start simpler."*

The AI is not offended. Redirect it. You'll learn more about shaping prompts intentionally in [Module 3: Getting Useful, Honest Output](../03-useful-honest-output/).
